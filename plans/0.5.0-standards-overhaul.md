# Spec: Standards overhaul → midas 0.5.0

Status: approved (docket ruled 2026-07-08)
Owner: Matt
Scope: midas repo only; midian-side findings become filed issues on midianio/midian.

## Why

An audit (2026-07-08) of the standards against their living reference implementation
(midian, pinned 0.4.1) found two classes of rot:

1. **Drift** — the standards prose still describes the mid-Go→Rust-port snapshot it was
   extracted from. In every dated disagreement, midian is more current than the standard.
2. **Enforcement dishonesty** — the registry claims 13 check-tier conventions but only 10
   execute. Three (`BE-0014`, `FE-0006`, `OPS-0003`) are wired to the `artifact-hash`
   check kind, which always returns `Skipped`. `BE-0012` is documented as "clippy denies
   print macros" but the executed mechanism is a regex grep. `FE-0001`'s check only
   asserts `src/lib/state` exists. This contradicts the README preamble's promise that
   the gate never claims a rule it doesn't run.

Additionally, eight settled midian practices have no convention coverage; four were ruled
in for codification, four go to backlog.

## Release plan

Single **0.5.0** release. One-tag invariant holds: `cli/Cargo.toml` version,
`registry/conventions.json` version, and frozen `registry/history/0.5.0.json` move
together under one tag. midian re-pins once, after resolving or ledgering the D4 issue
(else the newly-live artifact-hash check blocks its gate).

Known non-goals this round:

- No fixes inside midian (issues only).
- The in-flight `why-rust.md` work lands independently of this effort — but the prose
  sweep (T2) should fix its internal inconsistency ("The two mechanical, check-tier
  backend rules" followed by three IDs).
- Prefix≠layer naming (`OPS-`/`process`, `AGT-`/`agent`) stays as-is; IDs are stable.
